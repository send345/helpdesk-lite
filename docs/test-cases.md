# Тест-кейсы для HelpDesk Lite

## Заявки (tickets)

1. **Создание заявки** — POST /tickets с корректными данными → 200, заявка создана
2. **Создание заявки без title** → ошибка валидации
3. **Создание заявки без author_name** → ошибка валидации
4. **Получение списка заявок** — GET /tickets → 200, список
5. **Фильтр по статусу** — GET /tickets?status=new → только новые
6. **Фильтр по приоритету** — GET /tickets?priority=high → только high
7. **Получение одной заявки** — GET /tickets/1 → 200, данные
8. **Получение несуществующей заявки** — GET /tickets/999 → 404
9. **Изменение статуса** — PATCH /tickets/1 {"status":"resolved"} → статус изменился
10. **Изменение приоритета** — PATCH /tickets/1 {"priority":"critical"} → приоритет изменился

## Комментарии (comments)

11. **Добавление комментария** — POST /tickets/1/comments → 200
12. **Добавление пустого комментария** → 400
13. **Комментарий к несуществующей заявке** → 404
14. **Получение комментариев** — GET /tickets/1/comments → 200, список
